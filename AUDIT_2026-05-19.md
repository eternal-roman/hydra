## HIGH Severity (1)

### H1. Walk-forward metrics silently zeroed — field name mismatch
- **File:** `hydra_backtest_server.py:798`
- **Category:** Bug — silent data loss
- **Description:** `_research_lab_run()` constructs `FoldMetrics` using
  `getattr(m, "max_dd_pct", 0.0)` where `m` is a `BacktestMetrics`.
  `BacktestMetrics` defines the field as `max_drawdown_pct` (hydra_backtest.py:200),
  not `max_dd_pct`. The `getattr` fallback always returns `0.0`, silently
  zeroing max-drawdown in every walk-forward research lab result.
  Same pattern for `n_trades` (line 801: `getattr(m, "n_trades", getattr(m, "trade_count", 0))`) —
  `BacktestMetrics` uses `total_trades`, not `n_trades` or `trade_count`.
- **Impact:** Walk-forward statistical comparison (Wilcoxon test) lacks
  drawdown and trade-count data. Rigor gate `mc_max_dd_ci_acceptable`
  operates on zeros.
- **Fix:** Map attributes to `BacktestMetrics` field names:
  `max_dd_pct=m.max_drawdown_pct`, `n_trades=m.total_trades`.

## MEDIUM Severity (11)

### M1. Grok model name out of sync with brain
- **File:** `hydra_thesis_processor.py:53`
- **Category:** Stale model reference
- **Description:** Uses `"grok-4-reasoning"` while `hydra_brain.py:452`
  uses `"grok-4.20-0309-reasoning"`. Comment says "keep in lockstep."
- **Fix:** Align to whichever is the current xAI model ID.

### M2. Missing prompt caching on brain system prompts
- **File:** `hydra_brain.py` (QUANT_PROMPT, RISK_MANAGER_PROMPT)
- **Category:** Cost optimization
- **Description:** Large deterministic system prompts sent every tick
  without Anthropic `cache_control={"type": "ephemeral"}`. Wastes ~10%
  input token cost on repeated calls.
- **Fix:** Add `cache_control` to system message blocks in `_call_llm()`.

### M3. Nudge scheduler docstring backwards about default state
- **File:** `hydra_companions/nudge_scheduler.py:6`
- **Category:** Documentation bug
- **Description:** Says "HYDRA_COMPANION_NUDGES=1 must be set explicitly
  to enable" but `config.py:98-105` shows nudges are ON by default
  (opt-out via `=0`).
- **Fix:** Correct to "Kill switch: HYDRA_COMPANION_NUDGES=0 disables."

### M4. time.time() in interval timing (Windows 15ms resolution)
- **File:** `hydra_agent.py` (20+ locations)
- **Category:** Windows portability
- **Description:** Interval/duration checks use `time.time()` which has
  ~15ms resolution on Windows. Not safety-critical (monitoring/logging
  only) but causes tick jitter.
- **Fix:** Switch interval checks to `time.perf_counter()`; keep
  `time.time()` for wall-clock timestamps in journal/log.

### M5. Reviewer cost constants don't match published Opus pricing
- **File:** `hydra_reviewer.py:287-288`
- **Category:** Stale constants
- **Description:** Input $15/MTok, output $75/MTok — but published Opus
  pricing is $5/$25. Cost estimates inflate 3x.
- **Fix:** Verify current Anthropic pricing and update constants.

### M6. Shadow validator potential race on engine reference
- **File:** `hydra_shadow_validator.py:249-269`
- **Category:** Concurrency
- **Description:** `ingest_candle()` retrieves engine reference under lock,
  then calls `engine.tick()` after lock release. Another thread could clear
  engines between. Low probability (agent tick is sequential) but latent.
- **Fix:** Call `engine.tick()` inside the lock or store strong reference.

### M7. Brain fallback swallows partial analyst output
- **File:** `hydra_brain.py:798-805`
- **Category:** Error handling
- **Description:** When risk manager fails after analyst succeeds, the
  entire analyst reasoning is lost in the exception handler's fallback.
- **Fix:** Preserve partial output in fallback reason string.

### M8. Stream lifecycle — silent thread leak on hung process
- **File:** `hydra_streams.py:103-126`
- **Category:** Observability
- **Description:** `BaseStream.stop()` join timeout fires silently. Operator
  never learns reader thread is still alive.
- **Fix:** Log when `t.is_alive()` after join timeout.

### M9. Missing test: HYDRA_COMPANION_LIVE_EXECUTION default-off
- **File:** `tests/` (absent)
- **Category:** Test coverage gap — safety-critical
- **Description:** No test verifies that `live_execution_enabled()` returns
  `False` when env var is unset. This is the single most important safety
  gate in the companion subsystem.
- **Fix:** Add `test_companion_config_defaults.py`.

### M10. Companion daily cost counter race at UTC midnight
- **File:** `hydra_companions/coordinator.py:531-537`
- **Category:** Thread safety
- **Description:** `_maybe_rollover()` clears `_daily_trades` under
  `_cost_lock`, but `live_executor.py:44-51` reads it outside the lock.
  Race window at midnight could allow one extra trade past daily cap.
- **Fix:** Acquire `_cost_lock` in live_executor around cap check.

### M11. Companion memory budget enforced after persist
- **File:** `hydra_companions/memory.py:168-173`
- **Category:** Ordering
- **Description:** `_enforce_budget()` runs after `_persist()`, so file
  can briefly exceed the 4KB budget if a crash occurs between.
- **Fix:** Enforce budget before persisting.
